## Overview

This repository contains the **complete, end-to-end executable pipeline** for rice price forecasting across Indian states. The pipeline combines:

| Component | Role |
|-----------|------|
| **Quantum Genetic Algorithm (QGA)** | Feature selection from 43 candidate features |
| **Quantum Ant Colony Optimisation (QACO)** | Alternative feature selector (comparison) |
| **Quantum PSO (QPSO)** | XGBoost hyperparameter optimisation |
| **XGBoost** | Trend modelling |
| **LSTM** | Residual (error correction) modelling |

The final hybrid model produces **7-day, 14-day, and 30-day ahead** price forecasts.

## 🗂️ Data Setup

### Using your own data
Place your Excel file at the project root and update `DATA_FILE` in `configs/pipeline_config.yaml`:

```yaml
data:
  file: "Rice_merged_master.xlsx"   # path to your data file
```

**Required columns** (see `data/sample/DATA_FORMAT.md` for full schema):

| Column | Type | Description |
|--------|------|-------------|
| `STATE_KEY` | str | State identifier |
| `DATE_STD` | date | Observation date (DD/MM/YYYY) |
| `Modal Price` | float | Modal price (₹/quintal) |
| `Min Price` | float | Minimum price |
| `Max Price` | float | Maximum price |
| `T2M_MAX`, `T2M_MIN` | float | Max/Min 2m temperature (°C) — NASA POWER |
| `PRECTOTCORR` | float | Precipitation (mm/day) — NASA POWER |
| `RH2M` | float | Relative humidity (%) |
| `WS2M` | float | Wind speed at 2m (m/s) |
| `ALLSKY_SFC_SW_DWN` | float | Solar radiation (MJ/m²/day) |
| `PRODUCTION_TONNES` | float | Annual production |
| `AREA_HA` | float | Cultivated area (hectares) |
| `YIELD_TON_PER_HA` | float | Yield (t/ha) |

### Using the sample dataset
The `data/sample/sample_data.csv` file contains 500 anonymised rows for smoke-testing the pipeline without access to the full dataset:

```bash
# Run on sample data
python src/rice_price_forecasting.py --config configs/pipeline_config.yaml --sample
```

## 🔬 Pipeline Steps

Run the full pipeline end-to-end:

```bash
python src/rice_price_forecasting.py
```

Or run individual steps:

```python
from src.rice_price_forecasting import (
    compute_descriptive_stats,   # Step 1
    engineer_features,           # Step 3
    run_qga,                     # Step 6 — QGA feature selection
    run_qpso,                    # Step 7 — QPSO hyperparameter opt.
    train_hybrid,                # Step 8 — Hybrid XGBoost + LSTM
    evaluate_horizons,           # Step 9 — Multi-horizon evaluation
)
```

### Step-by-step description

| Step | Function | Output |
|------|----------|--------|
| 1 | `compute_descriptive_stats()` | Per-state statistics table (Skewness, Kurtosis, JB, Shapiro-Wilk) |
| 2 | `plot_acf_figure()`, `plot_periodogram()` | `Fig_ACF.png`, `Fig_Periodogram_DoubleLog.png` |
| 3 | `engineer_features()` | `X_full.parquet` — 43 features, log-transformed target |
| 4 | `split_statewise()`, `build_expanding_folds()` | `fold_indices.pkl` — 10-fold expanding CV |
| 5 | `internal_split_statewise()` | Internal train/val split for QFS + QPSO |
| 6 | `run_qga()` / `run_qaco()` | `features_QGA.txt`, `features_QACO.txt` |
| 7 | `run_qpso()` | `best_xgb_params.json` |
| 8 | `train_hybrid()` | Trained XGBoost + LSTM, test metrics |
| 9 | `evaluate_horizons()` | `horizon_results.csv`, `Fig_True_Horizon_Performance.png` |

## ⚙️ Configuration

All tunable parameters are centralised in `configs/pipeline_config.yaml`.  
**No code changes are needed** to adjust hyperparameter search bounds, fold counts, or file paths.

Key parameters:

```yaml
qga:
  population: 28
  generations: 18
  elite_k: 6
  lambda_feat: 0.02        # Feature-count penalty weight

qpso:
  n_particles: 20
  iterations: 30
  beta: 0.75               # Contraction-expansion coefficient

hybrid:
  seq_len: 14              # LSTM input window (days)
  horizons: [7, 14, 30]   # Forecast horizons (days ahead)
  lstm_units: [64, 32]
  dropout: 0.2
  epochs: 30
  batch_size: 256
  early_stopping_patience: 5
```
